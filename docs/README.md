## ğŸ“ Documentation Standards

### File Naming
- Use SCREAMING_SNAKE_CASE for important docs: `AI_GUIDELINES_CURRENT.md`
- Use descriptive names: `KAIZEN_STRATEGY_NOV_21.md` not `strategy.md`
- Include version for version-specific docs: `v2.6.5` suffix
- Include dates for session docs: `NOV_21` format

### Organization Principles
1. **Active > Archive** - Keep active docs in main folders, old versions in archive
2. **Purpose > Structure** - Group by purpose (gardens, sessions) not arbitrary categories
3. **Consolidate > Proliferate** - Merge similar docs, don't create duplicates
4. **Update > Create** - Update existing canonical docs rather than creating new versions

### Maintenance
- **Quarterly consolidation** - Archive old versions every 3 months
- **Version-specific archives** - Move docs to archive when version is superseded
- **Canonical documents** - Maintain single source of truth (e.g., `AI_GUIDELINES_CURRENT.md`)
- **Cross-references** - Update links when moving/archiving documents
