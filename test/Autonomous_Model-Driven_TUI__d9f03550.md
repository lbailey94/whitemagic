- **Native function calling for cloud models** — Claude, GPT-4o, and Gemini all support native tool/function calling APIs. Currently we parse `[tool:name(args)]` from text, which is fragile. Cloud models should use their native function calling format — far more reliable, structured, and supports parallel tool calls
- **Error recovery for malformed tool calls** — If the model produces a bad tool call (wrong args, unknown tool), retry with a hint instead of silently failing. Local models especially will hallucinate tool names

### High-impact for daily use

- **Git integration** — `/git status`, `/git diff`, `/git commit -m "..."`, live git status in the left panel. The AI can run git commands, but a dedicated panel + slash commands makes the workflow much smoother
- **File tree browser** — Navigable tree in the center or left panel. Arrow keys to browse, Enter to open. The AI can `list_dir` but you also want to *see* your project structure at a glance
- **Command history** — Up/Down arrow to recall previous inputs (like shell history). Essential for re-running commands or editing a previous prompt
- **Dedicated code search tool** — `search_code(query, path)` and `grep_files(pattern, path)` as first-class tools. Faster and more structured than `run_command(grep ...)`. Returns file:line:match tuples the AI can act on
- **Token/cost tracking** — Show tokens consumed per turn and running total. For cloud models, show estimated cost. Critical when using OpenRouter with paid models

### Quality of life

- **Markdown rendering in chat** — Render tables, headers, lists, and code blocks with syntax highlighting in the RichLog. Currently raw text
- **Project memory** — Auto-detect and load AGENTS.md, remember key files and conventions across sessions. WhiteMagic's memory system already does this — just need to wire it into the TUI's system prompt builder
- **Auto-complete** — Tab completion for file paths, tool names, and slash commands in the input field
- **Watch mode** — `/watch` command that auto-runs tests when files change, showing pass/fail in the tool stream panel

### Power features (nice to have)

- **Multi-model routing** — Use the existing [InferenceRouter](file:///home/lucas/Desktop/WHITEMAGIC/core/whitemagic/inference/router.py:177:0-582:90) to auto-route simple tasks to small local models and complex tasks to cloud. Saves money and latency
- **Concurrent tasks** — `/task <description>` spawns a background agentic loop while you keep chatting. Results appear in the tool stream when done
- **LSP integration** — Go-to-definition, find references, rename symbol. Would need a language server process per project. Significant work but transforms the experience
- **Inline code execution** — `> ```python ... ``` ` runs a snippet and shows output inline. Useful for quick experiments without creating a file

These are things **no other TUI has** because they come from WhiteMagic's cognitive infrastructure:
